# Features — Index

This folder contains one file per Entie feature. Read the file for the feature you're working on; ignore the rest.

When new features are added, drop in a new `.md` file and link it here.

## Conventions

Every feature file follows the same structure so they're predictable:

1. **What it is** — plain-language description
2. **How it works** — mechanics, edge cases
3. **Who uses it** — woman, partner, or both
4. **Why it matters** — the underlying user pain
5. **How to talk about it** — copy guidance, words to use, words to avoid
